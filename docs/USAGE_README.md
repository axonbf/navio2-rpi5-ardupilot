# Template Usage README

## Document Status

`current`

## Objective

Explain how to reuse `docs/` as a base to start a new project from scratch.

## Recommended Steps

1. Copy the contents of `docs/` to the documentation folder of the new project.
2. Rename the destination folder to `docs/` if that will be the main documentation name.
3. Complete these files first:
   - `README.md`
   - `DOCUMENTATION_INDEX.md`
   - `PROJECT_OBJECTIVE.md`
   - `SESSION_HISTORY.md`
4. Fill in the technical part afterwards:
   - `TECHNICAL_DESIGN.md`
   - `TECHNICAL_SETUP.md`
   - `TECHNICAL_ARCHITECTURE.md`
   - `IMPLEMENTATION_PLAN.md`
5. Keep `current_opencode_session.md` as a reference of the operational session format if the new environment uses the same mechanism.

## Suggested Startup Order

1. Define objective and scope in `PROJECT_OBJECTIVE.md`.
2. Register the initial session in `SESSION_HISTORY.md`.
3. Adjust rules and notes in `README.md` if the new project needs nuances.
4. Update `DOCUMENTATION_INDEX.md` to reflect any new or deleted files.
5. Start technical work in `TECHNICAL_DESIGN.md`.

## Adoption Checklist

- [ ] Project name updated.
- [ ] Objective and scope filled in.
- [ ] Session history initialized.
- [ ] Documentation rules reviewed.
- [ ] Technical documents prepared.
- [ ] Operational session references confirmed or adapted.

## Note

This template seeks to reuse structure and rules, not specific content from a previous project.
